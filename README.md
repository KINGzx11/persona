# 人格引擎 · Persona Engine

一个用 Python + 大模型搭的小型「人格 / 态度」引擎。

核心想法：一个人对某个对象的态度不是一次拍定的，而是一条条「链接（link）」慢慢攒出来的。攒下来的链接会形成他对这个对象的**固有认知**（体现在所有链接 rank 的总和上），并成为他下次再遇到这个对象时做反应的前提。

---

## 一、核心循环

```
情景 ─分析→ 链接 ─内化→ 固有认知(rank总和)
                                   │
                       再次遇到对象 │
                                   ▼
                            回溯该对象的链接
                                   │
                    ┌──────────────┴──────────────┐
              下意识反应(看rank总和)        进一步回溯(找具体链接/记忆)
                    └──────────────┬──────────────┘
                                   ▼
                              做出反应
                                   │
                         对这次情景再分析
                                   ▼
                            创建一条新链接 ─→（回到开头）
```

拆成六步：

1. **分析情景**：把一句情景拆成结构化链接 `object / relation / summary / rank`。
2. **内化**：把这条链接按对象归堆，追加进链接网，形成 / 更新对该对象的认知。
3. **固有认知 = rank 总和**：一个对象身上所有链接的 rank 加起来，就是此刻对它的整体倾向。
4. **再遇对象，回溯链接**：再次遇到这个对象时，先调出它名下的全部链接当前提。
5. **做出反应（两层）**：
   - **下意识反应**：直接看 rank 总和，给出快速倾向。
   - **进一步反应**：需要时再回溯具体链接 / 找记忆，做出更细的反应。
6. **再分析 → 新链接**：把这次「遇到对象」的经历再分析一遍，生成一条新链接写回，成为下一次的前提。

## 二、数据结构：链接网（link net）

所有数据存在一个 JSON 文件里。一个文件 = 一个人格 / 一个场景。

最小单位是一条**链接（link）**：

```json
{
  "object":   "中药",
  "relation": "亲身管用",
  "summary":  "吃中药把咳治好了",
  "rank":     "+1"
}
```

| 字段 | 含义 | 说明 |
|---|---|---|
| `object`   | 对象 | 这条链接关于谁 / 关于什么 |
| `relation` | 关系 | 这个人和该对象之间的一次关系 / 联系 |
| `summary`  | 概述 | 这次事件的一句话总结 |
| `rank`     | 评分 | 这次的倾向，目前只用最简单的 `+1` / `-1` |

很多条链接按对象归堆，就成了**链接网**：

```json
{
  "中药": [
    { "object": "中药", "relation": "管用",         "summary": "...", "rank": "+1" },
    { "object": "中药", "relation": "不认可",       "summary": "...", "rank": "-1" },
    { "object": "中药", "relation": "缺乏科学基础", "summary": "...", "rank": "-1" }
  ]
}
```

同一个对象上可以同时挂着正、负链接（又信又疑），引擎要把这份拧巴留住。

## 三、程序分块

```
情景 ─①分析+内化→ 链接网(纯数据) ─③求和+解说→ 总叙事
                          ▲  │
                          │  └─②再遇对象→回溯→反应
                          └─────（反应后再分析，写回新链接）
```

- **`creat_relaation.py` —— 分析 + 内化**（步骤 1-2、6）
  把输入的情景客观拆成一条链接，追加进链接网。只如实记录发生了什么。
- **`react.py` —— 反应器（待写，本阶段要补的核心）**（步骤 4-5）
  再次进入情景时：先编码出 object → 回溯该 object 的全部旧链接 → 先给「下意识反应」(看 rank 总和) →（需要时）再回溯具体链接做进一步反应 → 把这次经历再分析成新链接写回。
- **`creat_net.py` —— 求和 + 解说**（步骤 3）
  读链接网，挑一个对象，把它所有链接的 rank 加总，并把全部 relation 喂给模型，生成一段「此刻对该对象的整体态度」总叙事。

AI 只在两头出现：**分析（情景→链接）** 和 **解说 / 反应**。中间的链接网是纯数据，不让 AI 插手，保证可复现、可追溯。

## 四、怎么跑

环境：
```bash
python -m pip install openai
# Windows (PowerShell) 配置 DeepSeek key：
setx DEEPSEEK_API_KEY "你的key"
```
程序里通过 `os.getenv("DEEPSEEK_API_KEY")` 读取，走 DeepSeek 的 OpenAI 兼容接口（`base_url="https://api.deepseek.com"`，`model="deepseek-chat"`）。

运行顺序：
```bash
python creat_relaation.py   # 录入经历（分析+内化）
python react.py             # 再遇对象，做出反应（待写）
python creat_net.py         # 汇总对某对象的整体态度
```
运行时先问用哪个链接网文件（如 `中药.json`）——换文件 = 换人格 / 场景。

## 五、暂时挂起（先跑通最小闭环，不急做）

- rank 是否从 `+1 / -1` 升级成数值 strength。
- 「联系改变」：新情景是否回头改动旧链接的权重（目前只追加新链接，不改旧的）。
- 时间衰减、拧巴度量化、需求(needs)作为调节器 —— 见《人格引擎 · 核心机制》。
