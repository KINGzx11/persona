# 人格引擎 · Persona Engine

一个用 Python + 大模型搭的小型「人格 / 态度」引擎。

核心思想一句话：**一个人对某个对象的态度，不是一锤子拍出来的，而是一条条「链接（link）」慢慢攒出来的。** 攒下的链接会成为他下次遇到新情景时做出反应的「前提」，于是就有了认知惯性。

> 注意：这里我们刻意**不叫「记忆 / 卡片」，而叫「链接（link）」**。每条链接记录的是「这个人」和「某个对象」之间的一次关系（relation）。

---

## 一、数据长什么样：链接网（link net）

所有数据存在一个 JSON 文件里。一个文件 = 一个人格 / 一个场景。

最小单位是一条**链接（link）**：

```json
{
  "object":   "中药",
  "relation": "亲身管用",
  "summary":  "吃中药把咳治好了",
  "rank":     "+1"
}
```

| 字段 | 中文含义 | 说明 |
|---|---|---|
| `object`   | 对象 | 这条链接是关于谁 / 关于什么的 |
| `relation` | 关系 | 这个人和该对象之间发生的关系 / 联系 |
| `summary`  | 概述 | 这次事件的一句话总结 |
| `rank`     | 评分 | 这次的倾向，目前只用最简单的 `+1` / `-1` |

很多条链接按对象归堆，就成了**链接网（link net）**：

```json
{
  "中药": [
    { "object": "中药", "relation": "管用",         "summary": "...", "rank": "+1" },
    { "object": "中药", "relation": "不认可",       "summary": "...", "rank": "-1" },
    { "object": "中药", "relation": "缺乏科学基础", "summary": "...", "rank": "-1" }
  ]
}
```

**两条铁律：**
1. **只增不改（append-only）**：新经历只往后追加，从不抹掉旧的。
2. **矛盾共存**：「管用 +1」和「缺乏科学基础 -1」可以同时挂在同一个对象上——人本来就是拧巴的，引擎要把这份拧巴留住。

## 二、程序分三块（各管一摊）

设计上故意把「该让 AI 干的事」和「只是记账的事」分开：

```
   情景输入 ──①编码──▶ 链接网(纯数据，无AI) ──③解说──▶ 总叙事
                            ▲
                            │②反应（带着旧经验当前提）
```

### 1. `creat_relaation.py` —— 编码器（接待室）
把你输入的「情景」客观地拆成一条链接（object / relation / summary / rank），追加进链接网。
这一步只负责**如实记录发生了什么**，不带个人态度。

### 2. `react.py` —— 反应器（本次新增的核心）
进入一个**新情景**时：
1. 先把情景客观编码出 `object`；
2. **调出这个人过去对该 object 的全部旧链接，当作「前提」**；
3. 带着这个前提去反应——如果新情景和旧经验冲突，他会先**质疑、抵触**，而不是立刻全盘接受（这就是认知惯性）；
4. 把这次「情景反应」形成的新关系，再存回链接网，成为下一次的前提。

> 当前阶段惯性体现在**反应的措辞**里（演出来的），还没体现在**数字权重**里（算出来的）。`rank` 暂时只用 `+1 / -1`。

### 3. `creat_net.py` —— 解说员（汇总）
读取链接网，挑一个对象，把它身上所有链接的 `rank` 加总，并把全部 relation 喂给大模型，生成一段**总叙事**——也就是「这个人此刻对该对象到底是个什么整体态度」。

## 三、怎么跑起来

### 1. 环境准备
- 安装 Python 3（建议官方版，装时勾选 `Add Python to PATH`）。
- 安装依赖：
  ```bash
  python -m pip install openai
  ```
- 配置 DeepSeek 的 API Key（设为环境变量）：
  ```bash
  # Windows (PowerShell)
  setx DEEPSEEK_API_KEY "你的key"
  ```
  程序里通过 `os.getenv("DEEPSEEK_API_KEY")` 读取，走 DeepSeek 的 OpenAI 兼容接口：
  ```python
  client = OpenAI(api_key=os.getenv("DEEPSEEK_API_KEY"),
                  base_url="https://api.deepseek.com")
  # model = "deepseek-chat"
  ```

### 2. 运行顺序
```bash
# 第一步：往链接网里录入经历
python creat_relaation.py

# 第二步：让他对新情景做出（带惯性的）反应
python react.py

# 第三步：汇总他对某个对象的整体态度
python creat_net.py
```

运行时会先问你**用哪个链接网文件**（如 `中药.json`）——
**换一个文件名，就等于换一个人格 / 一个场景。**

## 四、设计原则备忘

- **链接，而非记忆**：存的是「人 ↔ 对象」的关系，不是一张张孤立卡片。
- **AI 只在两头出现**：编码（情景→relation）和解说/反应。中间的链接网是纯数据，不让 AI 插手，保证可复现、可追溯。
- **人格 = 一个可保存的链接网文件**：换文件 = 换人格。
- **先跑通，再变复杂**：当前是最朴素的 `+1 / -1` 求和。
